Writing a new adapter

An adapter is a small class that runs your agent once and normalizes the result into the stable AgentResponse contract. CrewAI, Microsoft AutoGen, the OpenAI Agents SDK, and LangGraph already have first-party adapters — writing your own follows the same shape.

The contract

Every adapter subclasses AgentAdapter and implements one method:

from agenteval.adapters.base import AgentAdapter, AgentResponse

class MyAdapter(AgentAdapter):
    def run(self, prompt: str, **kwargs) -> AgentResponse:
        # invoke your agent, measure latency, normalize the result
        ...

AgentResponse fields

FieldMeaning
outputThe agent's final answer text (required).
tool_callsNames of tools actually invoked.
nodes_firedOrdered execution path (agents/nodes/routes) for trajectory scoring.
prompt_tokens / completion_tokensProvider-reported token usage, when available.
cost_usdProvider-reported or computed cost in USD.
latency_msWall-clock time for the invocation.
retrieved_context / citationsFor RAG agents — see the core concepts and the README's RAG evaluation mode section.
rawJSON-safe original payload, preserved for debugging/dashboards.

Worked example: a minimal HTTP-based agent

import time
import requests
from agenteval.adapters.base import AgentAdapter, AgentResponse

class HttpAgentAdapter(AgentAdapter):
    def __init__(self, endpoint: str):
        self.endpoint = endpoint

    def run(self, prompt: str, **kwargs) -> AgentResponse:
        started = time.perf_counter()
        response = requests.post(self.endpoint, json={"prompt": prompt}, timeout=30)
        response.raise_for_status()
        data = response.json()
        latency_ms = (time.perf_counter() - started) * 1000.0

        return AgentResponse(
            output=data["answer"],
            tool_calls=data.get("tools_used", []),
            prompt_tokens=data.get("usage", {}).get("prompt_tokens"),
            completion_tokens=data.get("usage", {}).get("completion_tokens"),
            latency_ms=latency_ms,
            raw=data,
        )

Let provider/network failures raise rather than returning a successful-looking response — the runner records exceptions as agent_error, which is excluded from correctness/hallucination denominators but still fails the regression gate loudly.

Registering it

Point a registry entry at your adapter's import path (either use agenteval init to scaffold this, or write it directly):

agents:
  my_agent:
    adapter: my_project.adapters:HttpAgentAdapter
    repository:
      env_var: MY_AGENT_PATH
    golden_suite: tests/golden/my_agent.yaml
    baseline: baselines/my_agent.json
    runs_dir: runs

See adapters/langgraph.py in the repository for a fully worked example that normalizes a more complex, multi-node execution into the same contract.
