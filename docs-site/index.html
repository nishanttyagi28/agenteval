Getting Started

AgentEval is a CI-for-agents evaluation harness: run an agent against YAML golden suites, score five reliability metrics (plus optional RAG metrics), compare against a versioned baseline, and turn regressions into a reviewable CI decision.

Install

python -m pip install nishanttyagi-agenteval
agenteval --help

Install only the framework integration you need:

python -m pip install "nishanttyagi-agenteval[crewai]"
python -m pip install "nishanttyagi-agenteval[autogen]"
python -m pip install "nishanttyagi-agenteval[openai-agents]"

Scaffold a new project

agenteval init auto-detects your framework (CrewAI, LangGraph, AutoGen, or the OpenAI Agents SDK) and generates a registry entry, a sample golden suite, and a GitHub Actions workflow: 

cd my-agent-project
agenteval init --agent-name my_agent

Run your first evaluation

agenteval run --agent my_agent

Every scored run is compared against a versioned baseline (when one is configured) and produces a JSON report, a Markdown comparison, and — via agenteval report — a self-contained HTML report.

Next steps: see the CLI reference for every command, or the adapter guide to wire up your own agent framework.

Core concepts

  • Golden suite — a YAML file of test cases (prompt, ground truth, expected tools/trajectory).
  • Adapter — a small class that runs your agent and normalizes its output into AgentResponse.
  • Baseline — a versioned JSON snapshot of a previous run, used as the regression comparison point.
  • Gate — the pass/fail decision from comparing a current run against the baseline, enforced in CI.
